    Example

    Here’s a complete folder structure example for a real-world React + TDD-ready project using:

    • ✅ Vitest (or Jest)
    • ✅ React Testing Library
    • ✅ MSW (for mocking APIs)
    • ✅ Cypress (for end-to-end testing)

    Overview

    📦 Project Layout Example (Hybrid TDD Structure)

    my-app/
├── public/
├── src/
│   ├── components/
│   │   └── Button/
│   │       ├── Button.tsx
│   │       ├── Button.test.tsx          # ✅ Unit/component test
│   │       └── Button.types.ts
│   │
│   ├── hooks/
│   │   ├── useAuth.ts
│   │   └── useAuth.test.ts              # ✅ Hook unit/integration test
│   │
│   ├── features/
│   │   └── auth/
│   │       ├── LoginForm.tsx
│   │       ├── LoginForm.test.tsx       # ✅ Component test
│   │       ├── authSlice.ts             # Redux slice
│   │       ├── authSlice.test.ts        # ✅ Pure logic test
│   │       └── api.ts                   # RTK Query or Axios logic
│   │
│   ├── utils/
│   │   ├── validators.ts
│   │   └── validators.test.ts           # ✅ Pure function test
│   │
│   ├── App.tsx
│   └── main.tsx
│
├── tests/
│   ├── integration/
│   │   └── loginFlow.test.tsx          # ✅ TDD integration test
│   └── e2e/
│       └── login.cy.ts                 # ✅ E2E with Cypress
│
├── cypress/
│   ├── support/
│   │   └── commands.ts
│   └── fixtures/
│       └── user.json
│
├── mocks/
│   ├── handlers.ts                     # MSW API mocks
│   └── server.ts
│
├── .env
├── tsconfig.json
├── vite.config.ts                     # Or use Webpack config
├── vitest.config.ts                   # For unit/integration testing
├── cypress.config.ts
├── package.json
└── README.md

    🧪 Test Types Placement (Why This Works)

    Folder/File Purpose
    Button.test.tsx Unit test for button UI behavior
    useAuth.test.ts Integration test for hook + state
    authSlice.test.ts Pure reducer test (Chicago-style TDD)
    loginFlow.test.tsx Integration: form submit + API mock
    login.cy.ts Full E2E flow with real browser
    mocks/ API mocking (e.g., MSW for Vitest)

    🧪 Sample Test: Button.test.tsx

    import { render, screen, fireEvent } from "@testing-library/react";
import Button from "./Button";

test("calls onClick when clicked", () => {
  const onClick = vi.fn(); // or jest.fn() with Jest
  render(<Button onClick={onClick}>Click me</Button>);
  fireEvent.click(screen.getByText(/click me/i));
  expect(onClick).toHaveBeenCalledTimes(1);
});

    🧪 Sample Test: loginFlow.test.tsx

    import { render, screen, fireEvent } from "@testing-library/react";
import { rest } from "msw";
import { setupServer } from "msw/node";
import LoginForm from "@/features/auth/LoginForm";

const server = setupServer(
  rest.post("/api/login", (req, res, ctx) => {
    return res(ctx.status(200), ctx.json({ token: "123" }));
  })
);

beforeAll(() => server.listen());
afterEach(() => server.resetHandlers());
afterAll(() => server.close());

test("logs in user successfully", async () => {
  render(<LoginForm />);
  fireEvent.change(screen.getByLabelText(/email/i), {
    target: { value: "test@example.com" },
  });
  fireEvent.change(screen.getByLabelText(/password/i), {
    target: { value: "password123" },
  });
  fireEvent.click(screen.getByRole("button", { name: /login/i }));

  const successMsg = await screen.findByText(/welcome/i);
  expect(successMsg).toBeInTheDocument();
});

    🧪 Sample Cypress Test: login.cy.ts

    describe("Login Flow", () => {
  it("allows user to log in", () => {
    cy.visit("/login");
    cy.get("input[name=email]").type("test@example.com");
    cy.get("input[name=password]").type("password123");
    cy.get("button").contains("Login").click();
    cy.contains("Welcome, test@example.com").should("be.visible");
  });
});
