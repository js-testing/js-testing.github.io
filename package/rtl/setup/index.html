    Setup

    📦 Installation Vitest + Testing Tools

    #  Install Vitest + Testing Tools
npm install -D vitest jsdom @testing-library/react @testing-library/jest-dom @testing-library/user-event

# pnpm
pnpm add -D vitest jsdom @testing-library/react @testing-library/jest-dom @testing-library/user-event

# Install Redux Toolkit + React Redux
npm install @reduxjs/toolkit react-redux

# Install Ant Design
npm install antd

    npm install -D vitest jsdom @testing-library/react @testing-library/jest-dom @testing-library/user-event @types/jsdom
    🔍 Breakdown
    • pnpm add = npm install
    • -D = install as devDependencies

    You're installing:

    • vitest: the test runner
    • jsdom: simulates a browser environment
    • @testing-library/react: renders React components in tests
    • @testing-library/jest-dom: adds helpful matchers
    • @testing-library/user-event: simulates user interactions
    ✅ After This
    1. Make sure vite.config.js has the Vitest config.

    2. Create src/setupTests.js and import:

      import '@testing-library/jest-dom';

    3. Create a sample test (like App.test.jsx).

    4. Run tests with:

      pnpm test
pnpm test:watch

    ⚙️ Configure Vitest in

    import { defineConfig } from "vite";
import react from "@vitejs/plugin-react";

export default defineConfig({
    plugins: [react()],
    test: {
        environment: "jsdom",
        globals: true,
        setupFiles: "./src/setupTests.js",
    },
});

    Update your vite.config.ts to enable jsdom and testing globals:

    /// <reference types="vitest" />
import { defineConfig } from "vite";
import react from "@vitejs/plugin-react";

export default defineConfig({
    plugins: [react()],
    test: {
        environment: "jsdom",
        globals: true,
        setupFiles: "./src/setupTests.ts",
    },
});

    🧪 Create setupTests.ts

    This will load global matchers from jest-dom into Vitest.

    📁 src/setupTests.js
    import '@testing-library/jest-dom';

    This file runs before every test, like Jest's setupFilesAfterEnv.

    📁 src/setupTests.ts
    import "@testing-library/jest-dom";

    Check package.json scripts

    "scripts": {
    "dev": "vite",
    "build": "vite build",
    "preview": "vite preview",
    "test": "vitest",
    "test:watch": "vitest --watch"
}

    npm run test
npm run test:watch

    "scripts": {
    "dev": "vite",
    "build": "vite build",

    And then run them using pnpm:

    Action Command
    Start dev server pnpm dev
    Build the app pnpm build
    Preview build pnpm preview
    Run all tests pnpm test
    Run tests in watch mode pnpm test:watch

    ✅ No other changes needed — pnpm just replaces npm as your package manager.

    Question

    🤔 So why install @testing-library/jest-dom if we’re using Vitest, not Jest?

    ✅ Here's the answer:

    • @testing-library/jest-dom is not tied to Jest the test runner.
    • It’s a set of custom DOM matchers (like .toBeInTheDocument(), .toHaveTextContent(), etc.) that work with any testing framework — including Vitest.
    • It’s called jest-dom simply because it was originally made for Jest — but the matchers work as long as you're using expect-style assertions, which Vitest also supports.

    ✅ So yes, you should use @testing-library/jest-dom with Vitest!

    It adds useful matchers like:

    Matcher Example
    .toBeInTheDocument() expect(element).toBeInTheDocument()
    .toHaveTextContent() expect(button).toHaveTextContent('Submit')
    .toHaveAttribute() expect(img).toHaveAttribute('src', 'logo.png')

    These make tests more readable and user-focused.

    📦 Summary

    Package Purpose Works with Vitest?
    @testing-library/react Render and query React components ✅ Yes
    @testing-library/jest-dom Add custom DOM matchers to expect() ✅ Yes
    Vitest Test runner (like Jest) ✅ Yes
