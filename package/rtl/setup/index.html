    üîç Breakdown
    • pnpm add = npm install
    • -D = install as devDependencies

    You're installing:

    • vitest: the test runner
    • jsdom: simulates a browser environment
    • @testing-library/react: renders React components in tests
    • @testing-library/jest-dom: adds helpful matchers
    • @testing-library/user-event: simulates user interactions
    ‚úÖ After This
    1. Make sure vite.config.js has the Vitest config.

    2. Create src/setupTests.js and import:

      import '@testing-library/jest-dom';

    3. Create a sample test (like App.test.jsx).

    4. Run tests with:

      pnpm test
pnpm test:watch

    ‚öôÔ∏è Configure Vitest in

    Question

    ü§î So why install @testing-library/jest-dom if we‚Äôre using Vitest, not Jest?

    ‚úÖ Here's the answer:

    • @testing-library/jest-dom is not tied to Jest the test runner.
    • It‚Äôs a set of custom DOM matchers (like .toBeInTheDocument(), .toHaveTextContent(), etc.) that work with any testing framework ‚Äî including Vitest.
    • It‚Äôs called jest-dom simply because it was originally made for Jest ‚Äî but the matchers work as long as you're using expect-style assertions, which Vitest also supports.

    ‚úÖ So yes, you should use @testing-library/jest-dom with Vitest!

    It adds useful matchers like:

    Matcher Example
    .toBeInTheDocument() expect(element).toBeInTheDocument()
    .toHaveTextContent() expect(button).toHaveTextContent('Submit')
    .toHaveAttribute() expect(img).toHaveAttribute('src', 'logo.png')

    These make tests more readable and user-focused.
