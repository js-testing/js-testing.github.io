    üß™ Testing in React: Vitest, Jest & React Testing Library Explained

    Testing in modern React apps can be confusing when you're not sure which tool does what. This blog post explains the roles of Vitest, Jest, and React Testing Library (RTL) ‚Äî and how to use them effectively for unit testing, logic, and UI behavior.

    Both Vitest and Jest are JavaScript testing frameworks, mainly used for unit and integration testing.

    Success
    • Vitest/Jest = The test runners.
    • React Testing Library = Library to test React component behavior.
    • You often use both together to test React apps effectively.

    To test logic, functions, unit tests, async code, etc., you mainly use Vitest or Jest ‚Äî not React Testing Library, since RTL is focused on React UI behavior.

    üîç What's the Difference?

    Vitest vs Jest

    Feature Vitest Jest
    Test runner ‚úÖ Yes ‚úÖ Yes
    Speed üöÄ Fast with Vite üê¢ Slower for large projects
    TypeScript support üß† Excellent ‚úÖ Good
    Vite Integration üî• Built-in ‚ùå Manual setup required
    Community/Ecosystem üå± Growing üå≥ Mature

    Use Vitest for Vite-based projects, and Jest for broader or legacy setups.

    üß™ What Does React Testing Library Do?

    React Testing Library does not run tests ‚Äî it helps you test how users interact with the UI.

    You can use RTL with either Vitest or Jest. It focuses on:

    • Rendering components
    • Querying DOM elements
    • Simulating user actions
    • Verifying visible behavior

    üõ†Ô∏è Testing Examples

    ‚úÖ 1. Testing Logic (Unit Test)

    // utils/calc.js
export function add(a, b) {
  return a + b;
}

    // calc.test.js
import { describe, it, expect } from "vitest";
import { add } from "./calc";

describe("add", () => {
  it("adds two numbers", () => {
    expect(add(2, 3)).toBe(5);
  });
});

    ‚úÖ 2. Testing Async Functions (like API calls or timers)

    // utils/fetchData.js
export async function fetchData() {
  const res = await fetch("/api/data");
  return res.json();
}

    // fetchData.test.js
import { vi, describe, it, expect } from "vitest";

global.fetch = vi.fn(() =>
  Promise.resolve({
    json: () => Promise.resolve({ name: "Test" }),
  })
);

import { fetchData } from "./fetchData";

describe("fetchData", () => {
  it("returns mocked data", async () => {
    const data = await fetchData();
    expect(data).toEqual({ name: "Test" });
  });
});

    ‚úÖ 3. Testing React Component (UI)

    // MyComponent.jsx
export default function MyComponent() {
  return <h1>Hello World</h1>;
}

    // MyComponent.test.jsx
import { render, screen } from "@testing-library/react";
import { describe, it, expect } from "vitest";
import MyComponent from "./MyComponent";

describe("MyComponent", () => {
  it("renders a message", () => {
    render(<MyComponent />);
    expect(screen.getByText(/hello world/i)).toBeInTheDocument();
  });
});

    ‚úÖ Testing Logic Inside React Components

    You still use Vitest, but for logic-only parts (e.g., utility functions, custom hooks, state management functions), you don‚Äôt need React Testing Library.

    Example: Testing a custom hook

    // hooks/useCounter.js
import { useState } from "react";

export function useCounter() {
  const [count, setCount] = useState(0);
  const increment = () => setCount((c) => c + 1);
  return { count, increment };
}

    // hooks/useCounter.test.js
import { renderHook, act } from "@testing-library/react";
import { useCounter } from "./useCounter";

describe("useCounter", () => {
  it("increments the counter", () => {
    const { result } = renderHook(() => useCounter());
    act(() => {
      result.current.increment();
    });
    expect(result.current.count).toBe(1);
  });
});

    üõ†Ô∏è renderHook is part of@testing-library/react-hooks or comes from the main RTL in newer versions.

    üéØ Summary

    Test Type Tool(s) Purpose
    Unit tests Vitest / Jest Pure logic, math, utilities
    Async functions Vitest / Jest API calls, async logic
    UI components Vitest / Jest + RTL React rendering and behavior
    Custom Hooks RTL renderHook + Vitest Hook logic, state behavior

    &

    Type of Test Tools Used Example
    Pure Logic Vitest / Jest Math functions, string utilities
    Async Code Vitest / Jest + vi.mock API calls, delays
    Component UI Vitest / Jest + RTL Rendering, user interaction
    Custom Hooks RTL renderHook + Vitest State logic in hooks

    üîö Final Thoughts

    Use Vitest if you're using Vite. Use Jest if you're not. Use React Testing Library to test what the user sees and does. Together, they help you write confident, maintainable, and effective tests for your React app.
